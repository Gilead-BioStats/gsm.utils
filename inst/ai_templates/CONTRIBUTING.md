# Contributing

## Branching
- Default development branch: dev
- Use small, scoped branches (one theme per branch).

## Local checks
In R:
- devtools::document()
- devtools::test()
- devtools::check()

## Style
- Prefer readable code over clever code.
- Keep diffs small and reviewable.
- Separate formatting-only PRs from behavior changes.

## Security
- Do not commit secrets (tokens, keys, credentials).
- If you discover a vulnerability, follow SECURITY.md.
